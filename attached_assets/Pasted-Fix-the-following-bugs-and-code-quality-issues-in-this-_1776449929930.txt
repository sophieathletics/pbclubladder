## 1. Profile page fields initialize empty

File: apps/web/src/pages/profile.tsx

The fullName, phone, and sex fields use useState which captures undefined on first render before the player data loads. Fix this by adding a useEffect that syncs the form state once player data arrives:

useEffect(() => {
  if (player) {
    setFullName((player as any).fullName ?? "");
    setPhone((player as any).phone ?? "");
    setSex((player as any).sex ?? "");
  }
}, [player]);

## 9. Position swap is not wrapped in a database transaction

File: apps/api/src/lib/ladder.ts (swapLadderPositions function)

The 3-step position swap (set to -999, update winner, update loser) is not atomic. If the process crashes between steps, the ladder data is corrupted. Wrap all three UPDATE statements in a db.transaction() call:

await db.transaction(async (tx) => {
  await tx.update(ladderStandingsTable).set({ position: -999 }).where(...)
  await tx.update(ladderStandingsTable).set({ position: winnerPos, wins: ... }).where(...)
  await tx.update(ladderStandingsTable).set({ position: loserPos, losses: ... }).where(...)
});

Apply the same transaction wrapping to the inactivity drop swap in apps/api/src/routes/cron.ts.

## 10. Logout is a no-op — tokens remain valid after logout

File: apps/api/src/routes/auth.ts

The POST /auth/logout endpoint currently does nothing. Add a token revocation table to the database:

CREATE TABLE revoked_tokens (token_hash text PRIMARY KEY, revoked_at timestamp NOT NULL);

On logout, insert a hash of the token. In the auth middleware (lib/auth.ts), after verifying the token, check if its hash exists in revoked_tokens and reject with 401 if so. Clean up tokens older than 7 days (the JWT expiry) periodically or on each auth check.

Add a complete forgot-password / reset-password flow:
